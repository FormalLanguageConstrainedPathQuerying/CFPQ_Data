This guide can help you start working with CFPQ_Data.

We use the NetworkX MultiDiGraph to represent the labeled graph. To familiarize yourself with this representation, see NetworkX tutorial.

We use the Pyformlang CFG to represent the context-free grammar. To familiarize yourself with this representation, see Pyformlang usage.

All functions are documented on the Reference page.

GraphsÂ¶

You are able to get a synthetic or real-world graph with CFPQ_Data.

Create a synthetic graphÂ¶

You can create many different synthetic graphs by using functions from Graph's generators.

For example, let's create a one cycle graph, the edges of which are marked with the letter a.

import cfpq_data
cycle = cfpq_data.labeled_cycle_graph(5, edge_label="a")

Get a real graphÂ¶

You can get many different real-world graphs from CFPQ_Data Dataset.

And present these graphs in convenient formats by using functions from Graph's readwrite.

For example, let's get a generations ontology graph.

generations = cfpq_data.graph_from_dataset("generations")

Change labelsÂ¶

You can change the specified graph labels by using function cfpq_data.graphs.utils.change_edges() from Graph's utils.

new_cycle = cfpq_data.change_edges(cycle, {"a": "b"})

Now the labels a have changed to b.

GrammarsÂ¶

You can create your own context-free grammar (CFG) or use a predefined one. We now have three representations of CFG documented on the Grammars page:

  1. Classic

  2. Chomsky Normal Form

  3. Recursive State Machine

Create a context-free grammarÂ¶

You can create context-free grammar by using function cfpq_data.grammars.readwrite.cfg.cfg_from_text() from Grammar's readwrite.

cfg = cfpq_data.cfg_from_text("S -> a S b S | a b")

Use predefined context-free grammarÂ¶

CFPQ_Data has many predefined grammars documented on the Grammar's samples page.

brackets = cfpq_data.brackets
brackets.to_text()  # 'S -> a S b\nS -> a b\n'

Change terminalsÂ¶

You can change the specified grammar terminals by using function cfpq_data.grammars.utils.change_terminals.change_terminals_in_cfg() from Grammar's utils.

brackets1 = cfpq_data.change_terminals_in_cfg(brackets, {"a": "b", "b": "c"})
brackets1.to_text()  # 'S -> b S c\nS -> b c\n'
